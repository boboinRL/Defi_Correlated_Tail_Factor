TF Product Thesis
Why it exists

The problem

DeFi failures rarely arrive one factor at a time.

Liquidation crises emerge from linked mechanisms: price feeds lag, collateral gaps down, DEX depth disappears, gas rises, keepers stall, and governance controls become part of the response surface.

Fragmented analytics

Market risk and code risk are usually separated

Probability dashboards ignore contract paths; audit reports rarely quantify joint market conditions.

False precision

A single score hides the mechanism

Users need factor marginals, dependence, confidence intervals, model versions, and scenario assumptions.

Untraceable AI

LLMs can overclaim without evidence boundaries

AI should classify and explain supplied evidence, not invent probabilities, source locations, or exploitability.

Our answer

Statistical tail engine

Calibrated marginal probabilities plus factor dependence and Gaussian-copula Monte Carlo.

Contract-aware scenario builder

Search an Ethereum contract, map its protocol type to relevant factors, and select 1d / 7d / 30d horizons.

Evidence-backed audit agent

Verified source, proxy resolution, Slither, deterministic triage, GLM analyst review, and downloadable reports.

Probability definitions

Three distinct questions, kept separate

P(all selected factors within horizon)
P(any selected factor within horizon)
P(at least two selected factors within horizon)
  • Marginal probabilities are estimated independently for each factor and horizon.
  • Historical co-occurrence is converted into a shrunk dependence matrix.
  • A stabilized Gaussian copula simulates correlated factor states.
  • Monte Carlo seeds make repeated calculations reproducible.
  • Confidence intervals communicate sampling uncertainty instead of hiding it.

Probabilities, confidence intervals, correlations, validation metrics, and Monte Carlo outputs come from code and data.

TF Usage & Boundaries
What users can trust

How to use

One dashboard, two complementary workflows.

Tail-risk research

  • Search or select an Ethereum smart contract.
  • Choose 1d, 7d, or 30d.
  • Select linked factors manually or ask GLM for recommendations.
  • Adjust severity and dependence settings.
  • Read joint probability, confidence, bad-debt, queue, and recovery estimates.

Contract audit

  • Click Run Full Audit.
  • Inspect evidence-backed candidates and source excerpts.
  • Review GLM analyst and critic reasoning.
  • Download the hashed JSON report.
  • Move credible candidates into fork tests and manual verification.

Trust boundaries

Designed to provide

  • Reproducible scenario calculations
  • Versioned model and evidence metadata
  • Transparent uncertainty and warnings
  • Prioritized audit candidates
  • Traceable AI interpretation

Does not claim

  • Guaranteed future loss frequency
  • Industry-certified correlations
  • Automatic proof of exploitability
  • Complete formal verification
  • Investment or security guarantees

Local launch

cd "E:\DeFi\Tail-Factor"

node server.js

Then open http://localhost:3000. API keys remain in PowerShell environment variables and are never exposed to the frontend.

Product direction

The next maturity layer is dynamic validation: mainnet-fork exploit tests, deployed-state invariant checks, richer incident labels, and broader protocol coverage. The current product already provides the evidence and model-version foundation required to add those capabilities responsibly.
